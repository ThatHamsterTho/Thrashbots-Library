BimTi Thrashbots Controller & RF library
Used for TrashBots as a general library for the Controller PCB & RF communication.
RF_Handler/Controller_Transmitter/Controller_Transmitter.ino

This is an example of how to use the CTRL class to transmit data. The sketch sets the controller name to "BattleBotTeam__Tx". This is the name of the transmitter Then in the setup it calls the CTRL::begin() is called and the name of the receiver is passed to open a link to the receiver. It checks if any errors occured, which will be printed to the serial and continues to the loop. In the loop it uses the CTRL::transmitCtrlData() method to send the current position of all the hardware switches, bumpers, pots and joysticks.

#include <BimTi_Controller.h>
// RF24 Transmitter Code
CTRL Controller("BattleBotTeam__Tx");
/*****************************************************
* Setup Function, to run once *
*****************************************************/
void setup() {
Serial.begin(9600);
while (!Serial) {}
// some boards need to wait to ensure access to serial over USB
// Initialize the controller pins to input and output
InitControllerPins();
bool exitcode = Controller.begin("BattleBotTeam__Rx");
// if the controller.begin function returned an error catch it here.
if(!exitcode){
while(1);
}
}
/*****************************************************
* Loop Function, to run repeatedly *
*****************************************************/
void loop() {
// send all the current input positions to the receiver
Controller.transmitCtrlData();
} // loop
